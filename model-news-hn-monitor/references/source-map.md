# Source map

Use this source map when monitoring new model releases for speed **and** relative credibility.

## 1. Leak / early-signal layer

Use these for fast detection only. Do not treat a single source here as confirmed fact.

### X accounts
- `@testingcatalog`
- `@rowancheung`
- `@therundownai`
- Ben's Bites ecosystem accounts/newsletters

### What to look for
- hidden model IDs
- grey-rollout screenshots
- new pricing/model picker entries
- leaked UI labels
- OpenRouter appearing to expose a new model before a formal blog post

## 4. Distribution / usability layer

Track these to know when a model is not just announced, but actually available to try.
- OpenRouter

## Operating principle

Use multiple layers together:
- early signal discovers
- official source confirms
- HF/GitHub proves open-source reality
- distribution platforms prove practical availability

Do not rely on one channel alone. Compatibility with the user's principle: **兼听则明，偏听则暗**.
