## 平台特征
- 公开文章页面常带较重前端脚本与反爬限制；轻量阅读器经常只能拿到标题或壳页。
- 直接抓取原始 HTML 时，正文通常仍在 DOM 内，可优先尝试 `#js_content` / `.rich_media_content`。
- 这类页面未必需要登录才能读，但不适合高频重试或激进浏览器自动化。

## 有效模式
- 已知文章 URL 时，先尝试公开只读路径，再做正文抽取。
- 对正文提取，优先选择 `div#js_content`，其次 `div.rich_media_content`。
- 如果轻量 fetch 只返回标题与作者，不代表正文不存在，可能只是提取器抽错区块。

## 已知陷阱
- `r.jina.ai` / 通用 readability 对公众号文章不稳定，可能超时或只返回摘要壳。
- 不要为读公开正文一开始就走登录态浏览器 attach；先做低侵入读取。
- 遇到平台提示、空结果或超时，不要密集重试同一路径，避免触发额外风控。
